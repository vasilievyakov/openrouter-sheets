# Примеры использования

## Пример 1: Определение бренда в новости

**Промпт:** `Определи бренд, упомянутый в новости. Если бренд не найден, напиши "Не определен"`

**Результат:** В новой колонке будет список брендов для каждой новости.

## Пример 2: Извлечение ключевых слов

**Промпт:** `Извлеки 5 ключевых слов из этой новости, разделенных запятыми`

**Результат:** Колонка с ключевыми словами для каждой строки.

## Пример 3: Анализ тональности

**Промпт:** `Оцени тональность текста: позитивная, нейтральная или негативная. Ответь одним словом.`

**Результат:** Колонка с оценкой тональности.

## Пример 4: Перевод

**Промпт:** `Переведи этот текст на английский язык`

**Результат:** Переведенные тексты в новой колонке.

## Пример 5: Извлечение даты

**Промпт:** `Извлеки дату из текста в формате ДД.ММ.ГГГГ. Если дата не найдена, напиши "Не указана"`

**Результат:** Колонка с датами.

## Пример 6: Краткое резюме

**Промпт:** `Напиши краткое резюме этого текста в 2-3 предложениях`

**Результат:** Краткие резюме для каждой новости.

## Пример 7: Категоризация

**Промпт:** `Отнеси эту новость к одной из категорий: Технологии, Спорт, Политика, Экономика, Культура, Другое`

**Результат:** Категории для каждой новости.

## Рекомендации по промптам

1. **Будьте конкретны** — четко указывайте, что нужно извлечь или проанализировать
2. **Задавайте формат ответа** — укажите, в каком формате нужен ответ (одно слово, список, дата и т.д.)
3. **Используйте примеры** — если нужен специфический формат, укажите пример в промпте
4. **Обрабатывайте отсутствие данных** — просите возвращать конкретное значение, если данных нет
5. **Ограничивайте длину** — если нужен краткий ответ, укажите это в промпте

## Оптимизация для больших объемов

Для обработки больших таблиц (4000+ строк):

1. **Тестируйте на малых данных** — сначала протестируйте на 10-20 строках
2. **Проверяйте квоты** — убедитесь, что у вас достаточно токенов OpenRouter
3. **Мониторьте GitHub Actions** — следите за прогрессом в логах
4. **Используйте кэш** — скрипт автоматически кэширует одинаковые запросы

## Отладка

Если результаты неожиданные:

1. Проверьте промпт — он должен быть понятным для модели
2. Посмотрите примеры результатов — найдите проблемные строки
3. Уточните промпт — добавьте больше инструкций или примеров
4. Проверьте логи GitHub Actions — там могут быть ошибки API
